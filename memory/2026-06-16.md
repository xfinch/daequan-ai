# 2026-06-16 - Comcast CRM Contact Parser Foundation

## Durable Executable: Contact Name Parser

**Status:** Planned, awaiting implementation

**Purpose:** Parse contact names with roles/designations into structured fields for gatekeepers and decision makers. Built as a permanent, reusable, testable module.

---

## Requirements

### New Database Columns (schema.sql)
- `gatekeeper_first_name` - Gatekeeper's first name
- `gatekeeper_last_name` - Gatekeeper's last name  
- `decision_maker_first_name` - Decision maker's first name
- `decision_maker_last_name` - Decision maker's last name
- `other_contacts` - JSON array for additional people

### Parsing Rules

**Role Mapping:**
- **Gatekeeper:** "gatekeeper", "gk", "receptionist", "receptionist/gatekeeper"
- **Decision Maker:** "owner", "dm", "decision maker"
- **Other:** All other roles (manager, artist, etc.) → stored in other_contacts with role noted

**Name Parsing:**
- Standard: "John Smith" → first: "John", last: "Smith"
- **Dr. Exception:** "Dr. John Smith" → first: "Dr. John", last: "Smith"
- Single name: "John" → first: "John", last: ""

**Separators:** comma, slash, "&"

**Example:**
- Input: `Andre (owner), Jordan (artist on duty)`
- Output:
  - `decision_maker_first_name`: "Andre"
  - `decision_maker_last_name`: ""
  - `other_contacts`: `[{"first_name": "Jordan", "last_name": "", "role": "artist on duty", "type": "other"}]`
  - Notes appended: "Contact roles: Andre (owner), Jordan (artist on duty)"
